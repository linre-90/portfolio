Documentation

This documentation provides simple usage instructions for portfolio API. It can be connected from external resources, like mobile app, with authentication. API is very limited and restricted only for certain usages.

API Documentation

BaseUrl: https://www.{some_domain_name}.{some_domain_extension}/contact/api/v1

  • Authentication
  • GET methods
  • DELETE methods
  • UPDATE methods
Authentication

User authenticates to api using endpoint "BaseUrl"/login. Provide valid username and password in request body. Then User receives authorization cookie and session.

GET endpoints

BaseUrl request returns all contacts in json format.

BaseUrl/{id} request fetches single contact from server based on url or returns 404 if not found

DELETE endpoints

BaseUrl/{id} delete request removes parameter id corresponding entry from database.

UPDATE endpoints

BaseUrl/{id} update request toggles contact read value to !contact read value.

Accessibility

All pages beyond /admin requires authentication with correct password and username. User is provided login form in BaseUrl/login and after that cookies are set for admin usage.

Functionality

Admin pages provide customized crud operations, ability ro read contacts and documentation for API usage.
